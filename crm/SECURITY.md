# Security Policy

## Overview
This project is a personal, non‑commercial tool designed to run entirely in the user’s browser and store data exclusively in the user’s own Google Drive account. The maintainer does not operate any servers, does not collect data, and has no access to user information.

This document outlines the security expectations, responsibilities, and limitations associated with using this software.

---

## Supported Versions
This project is maintained on a voluntary, best‑effort basis.  
There are **no guaranteed release schedules**, **no service‑level commitments**, and **no formal support obligations**.

Security updates may be provided when possible, but the maintainer is not required to address every issue.

---

## Reporting a Vulnerability
If you discover a potential security issue:

- Please open a GitHub Issue **without including personal or sensitive data**, or  
- Contact the maintainer privately if the report contains confidential information.

Reports will be reviewed on a best‑effort basis.  
There is **no guaranteed response time**.

## Security Expectations for Users
To reduce risks, users should:

- Use their own Google OAuth credentials and restrict them to trusted domains  
- Review and understand the permissions requested by the application  
- Avoid storing sensitive or regulated personal data (e.g., health, financial, or special‑category data)  
- Keep their browser and operating system updated  
- Use the application only in trusted environments  
- Regularly review the contents of their Google Drive
